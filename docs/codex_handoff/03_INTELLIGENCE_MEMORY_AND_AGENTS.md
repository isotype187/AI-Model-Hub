# Nexus98 Codex Handoff 03
# Intelligence Memory and Agents

## Purpose

Define the intelligence layer.

## Model Intelligence

Nexus98 may automatically select:

- model
- provider
- agent
- strategy

Automatic choices must be visible.

User override remains available.

## Strategy System

Strategies influence:

- reasoning style
- tool selection
- execution approach
- validation requirements

Examples:

- documentation first
- conservative coding
- deep analysis
- testing focused
- security focused

## Agent System

Agents are specialized workers.

Agents contain:

- purpose
- capabilities
- tools
- permissions
- memory

## Memory

Memory should be:

- database backed
- searchable
- lifecycle managed
- compressed when appropriate

Avoid:

- checkpoint sprawl
- duplicate history

## Code Memory

Track:

- modules
- files
- dependencies
- relationships
- decisions

Purpose:

safe understanding and modification.
